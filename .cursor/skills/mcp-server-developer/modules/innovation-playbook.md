## 2. Ideation Framework

1. **Identify unmet workflow** ‚Äì monitor community forums, Discord, GitHub issues for recurring pain points.
2. **Inventory data/API assets** ‚Äì list accessible APIs (official or scraped), datasets, or local tools.
3. **Map MVP operations** ‚Äì describe the minimal operation set (list, fetch, analyze, summarize).
4. **Assess risk/compliance** ‚Äì ensure usage respects API terms, privacy, and security.
5. **Prototype quickly** ‚Äì build service stubs, simulate responses, validate user value before full integrations.

## 3. Experience Design Tips

- Provide step-by-step guidance via tool responses (header, context, next actions).
- Return structured JSON with clearly labeled sections (`summary`, `actions`, `references`).
- Offer follow-up suggestions (‚ÄúRun `plan`, `book`, `gather_docs` next‚Äù).
- Include disclaimers where data may be stale; point to refresh commands.
- Design for graceful degradation if external APIs fail (fallback copy, cached data).

- Encourage community contributions (issues, PRs, plugin expansions).
